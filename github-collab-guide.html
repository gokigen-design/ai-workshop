GitHub 共同編集ガイド

クライアントと
同じファイルを
一緒にいじる方法

GitHubを使えば、離れた場所にいる人と同じファイルを安全に編集できます。このガイドで全体の流れを掴みましょう。

01 — GitHubとは

GitHubは
「インターネット上の共有フォルダ」

Dropboxや共有Googleドライブに近いイメージです。ただしファイルの「変更履歴」も全部残ります。

普通のフォルダと同じこと
ファイルを入れる・取り出す・編集する。基本的な操作はフォルダと同じです。
変更履歴が残る
「誰が・いつ・何を変えたか」がすべて記録されます。間違えても前の状態に戻せます。
複数人で使える
Collaborator（共同作業者）として招待した人だけが編集できます。
無料で使える
基本機能は無料。プライベート（非公開）リポジトリも無料で作れます。
合言葉：「GitHubにpushする」＝「保存してみんなと共有する」
「GitHubからcloneする」＝「共有フォルダを自分のPCにコピーする」
02 — 招待する側（あや）の手順

一緒に編集したい人を
招待する

GitHubの設定画面から相手を招待するだけ。ここはブラウザ操作のみで完結します。

あやがやること
1
GitHubでリポジトリを開く
github.com → 対象のリポジトリ（例：gokigen-design/ai-workshop）を開く
2
Settings → Collaborators を開く
上部タブの「Settings」→ 左メニュー「Collaborators（協力者）」をクリック
3
「人を追加する」→ ユーザー名を入力
相手のGitHubユーザー名を入力して選択 → 「Add」をクリック
4
相手に「招待メール送ったよ」と連絡する
GitHubから招待メールが自動で届きます。LINEで一言「承認してね」と送るとスムーズ
03 — 招待された側（クライアント）の手順

招待を受け取ったら
自分のPCにコピーする

「clone（クローン）」とは、GitHubのフォルダを自分のパソコンにコピーすることです。1回やるだけです。

クライアント・チームがやること
1
招待メールを承認する
GitHubからのメールを開いて「Accept invitation」をクリック
2
ターミナルを開いてcloneする
以下のコマンドをコピペして実行。フォルダが自動で作られます。
# 共有フォルダを自分のPCにコピーする（1回だけ）
git clone https://github.com/gokigen-design/ai-workshop.git
3
フォルダに移動してClaude Codeを起動
cloneしたフォルダの中に入って、Claude Codeを起動します
# フォルダに入る
cd ai-workshop
# Claude Codeを起動
claude
04 — 編集して保存・共有する

変更したら
「push」で全員に届ける

Claude Codeで編集したら、pushでGitHubに送ります。これが「保存して共有」の操作です。

あや・クライアント共通
# 変更したファイルを指定（preparation.htmlの場合）
git add preparation.html

# 「何を変えたか」のメモをつけて記録
git commit -m "〇〇を修正した"

# GitHubに送って全員と共有
git push
覚え方：
add＝「このファイルを持っていく」
commit＝「メモと一緒にカバンに詰める」
push＝「GitHubに送る」
05 — まとめ

コマンド早見表

この4つだけ覚えれば大丈夫。迷ったらここに戻ってきてください。

git clone
GitHubのフォルダを自分のPCにコピーする（初回1回だけ）
git add
保存するファイルを選ぶ
git commit
「何を変えたか」のメモをつけて記録する
git push
GitHubに送って全員と共有する← 一番大事
注意点：同時に同じファイルを編集するとぶつかる（コンフリクト）ことがあります。
「私が今編集するね」とLINEで一言声かけし合うのがベストです。
