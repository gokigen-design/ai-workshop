受講生のみなさん
LPとは
そもそも、ランディングページって何？

「LPを作る」と言われても、何を作るのか・なぜ必要なのか、ピンと来ていない方も多いはず。まずはここを揃えてから、自分のサービスをLPに落とし込んでいきましょう。

1
LP＝「1つのゴール」を達成するための1ページ

LP（ランディングページ）とは、訪問してきた人にたった1つの行動を取ってもらうために作られた、縦長の1ページのこと。
申込み・LINE登録・お問い合わせ・予約──ゴールはひとつ、それに向かって全ての要素が連携しています。

HOMEPAGE
ホームページ
複数ページに情報を網のように広げる。会社の名刺。
LANDING PAGE
LP（ランディングページ）
1ページに動線を集中。申込みへの一本道。
2
アートとデザインは違う
Art / アート
自己表現。作者が伝えたいことを形にする。「好き」「美しい」が基準。見る人の解釈に委ねる。
Design / デザイン
目的達成のための設計。見る人に「行動してもらう」ことが基準。綺麗かどうかは手段でしかない。

LPは「デザイン」です。「おしゃれに見せたい」は手段。「申し込んでもらう」「問い合わせしてもらう」が目的です。

3
情報をデザインする3つの原則

LPに載せるのは「全部」ではなく「必要なものだけ」。情報の取捨選択こそがデザインの第一歩です。

1
このLPで何をしてほしいか、1つ決める
「申し込んでほしい」「問い合わせしてほしい」「LINEに登録してほしい」。ゴールは必ず1つ。複数あると何もしてもらえません。
2
誰に届けたいかを1人に絞る
「全員に刺さるLP」は誰にも刺さりません。理想の1人を思い浮かべて書くと、読んだ人が「これ私のことだ」と感じます。
3
載せない情報を決める勇気を持つ
「全部説明しなきゃ」は誰でも陥る罠。情報が多いほど読まれません。優先順位を決めて削るのもデザインの仕事です。
4
LPの基本構造（黄金パターン）

LPには「読み手を行動に導く」ための型があります。順番が大事で、これを外すと途中で読み手が離脱します。

  1. 01
    FV（ファーストビュー）：最初の3秒で「自分のことだ」と思わせる
  2. 02
    悩み（共感）：「そうそう、それで困ってる」と頷かせる
  3. 03
    解決策（あなたのサービス）：その悩みをこう解決します、と提示
  4. 04
    得られる未来：このサービスを受けたらこうなる、を見せる
  5. 05
    講師紹介・実績：「この人なら大丈夫そう」と信頼を積む
  6. 06
    料金・FAQ：申込み前の不安をすべて潰す
  7. 07
    CTA（申込みボタン）：1つのゴールへ
5
準備が9割：Claudeへの指示の質＝LPの質

どんなにきれいなLPでも、伝えるべき情報が整っていなければ申込にはつながりません。Claudeはあなたの指示通りに作るので、指示の質がそのままLPの質になります。

トークンって何？

ClaudeのチャットとClaude Codeはトークンを共有しています。チャットで何度も壁打ちを繰り返すと、あっという間に上限に達してしまいます。LP制作に入る前に、AIの使い分けを覚えておきましょう。

あとはClaude Codeで細かく調整
「色を変えて」「この文章を直して」など、具体的な修正はClaude Codeに。テキスト量が少ない修正指示はトークンをほとんど消費しない。

─ Claude Code の使い方
「フォルダ」って何？どこに作ればいい？

Claude Code は「指定したフォルダの中だけ」を見るしくみです。PC全体を見られないように、作業場所を1つ決めてあげるのがフォルダ選択です。

おすすめ：デスクトップに「クラウドコード」というフォルダを作り、その中に案件ごとのサブフォルダを作る。
例：クラウドコード / LP制作 / 6月6日ワークショップ

素材ファイル（写真・参考資料など）も、Claude に見せたければそのフォルダにコピーして入れておきましょう（移動ではなくコピー）。

「許可を求められた」のダイアログ、Yes押して大丈夫？

自分の作業フォルダの中なら全部 Yes でOKです。Claude は安全のため、ファイルを書き換える前に毎回確認してきます。慣れてきたら「このセッションの間は許可」を選ぶと聞かれなくなります。

不安な時はそのまま読んで判断すれば大丈夫。怪しい操作（パスワードや決済まわり）でない限り、Yesで進めて問題ありません。

Pro と Max、どっちのプランがいい？

ワークショップは Pro（$22／月・税込）でOKです。Pro で十分LPは作れます。

Max（$100）は1日に何時間も Claude Code を使い倒したい人向け。回数制限に当たって困るようになったらアップグレードを検討してください（いつでも変更可）。

「Sonnet」「Opus」ってモデル選択、どっちにすればいい？

基本は Sonnet で大丈夫です。動作も速くて、LP制作にはこれで十分。

「もっとデザインにこだわりたい」「複雑な指示を理解させたい」時だけ Opus に切り替えてみてください。Opus は賢い分、消費が大きいので使い分けがおすすめ。

既存ファイルや素材を、新しいフォルダに「移動」していい？

移動ではなく「コピー」してください。元のファイルはそのままに、コピーをClaude用フォルダに入れる。これで「あ、消えちゃった！」を防げます。

スマホからでもできる？

パソコンを強くおすすめします。Claude Desktop アプリも Claude Code 機能もパソコン用です。スマホでは Claude のチャットアプリは使えますが、ファイル作成などの本格的な作業はできません。

Windows の「Sモード」は、Microsoft Store 以外のアプリをインストールできない制限モードです。

スマホとPCでログイン方法が異なると、別アカウントとして認識されます。

スマホで使っているログイン方法（Google / Apple / メールアドレス）を確認して、PCでもまったく同じ方法でログインしてください。どの方法でログインしたか分からない場合は Slack で教えてください。

■ コピー（文章）のルール
━━━━━━━━━━━━━━━━━━━━━━

【禁止するコピー表現】
「あなたの可能性を広げる」「新しい一歩を」「理想の〇〇へ」
「本当の自分を取り戻す」「すべてが変わる体験」
のような、中身のない抽象的なコピーは一切使わないこと。

【禁止するAI定型句】
「さあ、始めましょう」「いかがでしょうか？」「〜の世界へようこそ」
「革新的な〜」「今こそ〜のとき」は使わないこと。

【代わりにやること】
・数字を入れる（「90分で完成」「参加者80名」「週3時間の節約」など）
・固有名詞を入れる（ツール名・サービス名・地名・具体的なテーマ）
・動詞から始める（「作る」「設定する」「確認する」など）
・情報が足りない箇所は勝手に作らず「ここに〇〇を入れてください」とプレースホルダーにする

【絶対禁止】
架空のお客様の声・人物名・実績数字（「満足度97%」など）は作らないこと。
実績セクションが必要な場合は「ここにお客様の声を入れます」とダミーで置くこと。

━━━━━━━━━━━━━━━━━━━━━━
■ デザインのルール
━━━━━━━━━━━━━━━━━━━━━━

【絵文字】
✨ 🔥 💡 🎯 💪 🚀 ⭐ 📌 🎉 👑 💫 などの絵文字は一切使わないこと。
チェックマークやリスト装飾はCSSの疑似要素（::before）や左ボーダーで実装すること。

【フォント】
・書体は最大3種類まで。4種類以上は使わないこと。
・明朝体を長い本文に使わないこと。

【フォントサイズ】
・本文：16px以上（14px以下は禁止）
・大見出し（H1）：font-size: clamp(28px, 7vw, 48px)
・中見出し（H2）：font-size: clamp(22px, 5vw, 32px)
・小見出し（H3）：font-size: clamp(17px, 4vw, 22px)
・セクションラベル：11px
・注釈・補足：12px
・PCとスマホで同じサイズのままにしないこと（clampを使うこと）

【色】
・アクセントカラーは1色だけ使う。3色以上は禁止。
・色はCSS変数（:root）で管理すること。
・テキスト：#1e1e1e
・補足テキスト：#6a6a6a
・背景は純白（#ffffff）ではなく、#faf8f5 など少し色味のある白にする。
・原色（red, blue, greenなど）をそのまま使わないこと。
・派手なグラデーションは禁止。使う場合は同系色の微細な変化のみ。

【レイアウト・余白】
・セクション間の余白：PC 64px / スマホ 48px
・要素間の余白は8の倍数で統一（8, 16, 24, 32px）
・左右パディングは最低20px確保すること（スマホで端が切れないように）
・テキストに横幅の制限を入れること（max-width推奨）
・シャドウは薄く広く：box-shadow: 0 8px 30px rgba(0,0,0,0.05)
・角丸は8pxか16pxに統一すること。要素ごとにバラバラにしない。

【行間】
・本文の行間：1.7〜1.8
・見出しの行間：1.3〜1.4

画像形式の使い分け（JPG / PNG / WebP）

画像はLP全体の容量の大半を占めます。目安は1枚500KB以下。形式を正しく選ぶだけで大きく軽くなります。

形式 向いている用途 特徴
JPG 写真・背景画像 軽量。透過不可。
PNG ロゴ・アイコン・切り抜き画像 透過OK。写真には重くなる。
WebP どちらでもOK（推奨） 軽さと透過を両立。最新ブラウザ向け。

─ 環境構築でつまずいた時
Mac で「xcode-select --install」というエラーが出ました

Mac に必要な開発ツール（Xcode Command Line Tools）が入っていないだけです。

1. ⌘ + space →「ターミナル」と入力して開く
2. xcode-select --install をコピペして Enter
3. ポップアップで「インストール」をクリック → 完了したら Claude Code を再起動

解決しない時はエラー画面をスクショ（⌘ + shift + 4）して Claude のチャットに貼り付けて聞いてください。

Windows で「Git をインストールしてください」と出ました

Claude Code は内部で Git というソフトを使うので、先に Git for Windows をインストールする必要があります。

1. git-scm.com/downloads/win から「Click here to download」をクリック
2. ダウンロードしたファイルを開いて、すべて「Next」を押し続けて Install → Finish
3. Claude Code を再起動

Google アカウントでログインできない

メールアドレスで新規アカウントを作成してください。Claude のアカウントは Google ログインではなくメール登録が確実です。

Desktop アプリが開かない / Windows でブロックされる

PC を再起動してからもう一度試してください。Windows で「WindowsによってPCが保護されました」と出たら、「詳細情報」→「実行」をクリックすれば進めます。

─ Claude Code の使い方
「フォルダ」って何？どこに作ればいい？

Claude Code は「指定したフォルダの中だけ」を見るしくみです。PC全体を見られないように、作業場所を1つ決めてあげるのがフォルダ選択です。

Claudeの利用制限は、Anthropic公式ヘルプによると「直近5時間のローリングウィンドウ方式」で判定されています。 つまり直近5時間の使用履歴が残り続ける仕組みです。

制限にひっかかる主な理由
  1. 5時間のローリングウィンドウ（公式）

  2. 週次制限（Weekly Cap）
