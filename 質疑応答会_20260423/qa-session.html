HTMLファイルについて

Claude Codeの中でも今後よく出てくる、HTMLファイルについての先行説明です。「index.html」って何？ タグって何？ 身近なファイルに例えながら、ゆっくりいきます。

1-3そもそもHTMLファイルって何？

「HTMLファイル」「index.html」という言葉がワークショップでよく出てきますが、そもそも何のことなのか。
身近なファイルに例えながら、ゆっくり説明します。

今あなたが見ているこのページも、HTMLファイルです

Chromeで開くサイト、LINEのリンクから開くページ、会社のホームページ……
これらは全部、「HTMLファイル」という1つのファイルを、ブラウザ（Chrome／Safari）が読み取って表示しているだけ。
特別なものではなく、Wordのファイルと同じく「ただのファイル」です。

身近なファイルで例えると

DOC
Word文書
.docx → Microsoft Word
JPG
写真
.jpg / .png → 写真アプリ・プレビュー
PDF
資料
.pdf → プレビュー・Acrobat
HTML
Webページ
.html → Chrome / Safari（ブラウザ）
つまりHTMLファイルとは
ブラウザで見るための文章ファイル。ただそれだけ。
Wordを開けばWord文書が表示されるように、ブラウザを開けばHTMLファイルが表示されます。

中身は「ただの文字の集まり」です

HTMLファイルを開くと、こんな文字がズラーっと書いてあります。
左の文字（コード）をブラウザが読み取って、右のような見た目のページを作っています。

<!DOCTYPE html> <html> <body> <h1>あなたらしい LPを作ろう</h1> <p>申し込みは こちらから。</p> </body> </html>
● ● ●　localhost/index.html

あなたらしい
LPを作ろう

申し込みはこちらから。

<h1> とか <p> って何？

左の文字の中にある <h1>、<p> などが「タグ」と呼ばれるもの。
「ここは見出し」「ここは段落」と書いてある付箋・ラベルのような目印です。

タグ 意味 例えるなら
<h1> 〜 </h1> 大見出し 付箋「ここは大きなタイトル」
<p> 〜 </p> 段落（普通の文章） 付箋「ここは本文」
<img> 画像を表示 「ここに写真を置いて」
<a> 〜 </a> リンク 「タップすると別ページへ」

ブラウザはこのタグ（目印）を見て、
「じゃあ大見出しは大きく表示しよう」「画像はここに貼ろう」と判断し、
キレイな見た目のページを作ります。

「index.html」って何？

ワークショップで作るLPは、最終的に index.html という1つのファイルになります。
index（インデックス）は「入り口・目次」の意味。
サイトを開いたときに最初に表示されるページには、伝統的に index.html という名前を付ける習慣があります。

ここまでのまとめ
  • Webページ＝HTMLファイル（拡張子 .html）
  • ダブルクリックするとブラウザ（Chrome / Safari）で開ける
  • 中身は <タグ> だらけの文字の集まり
  • タグは「ここは見出し」「ここは画像」という目印
  • LPのファイル名はたいてい index.html
ここでClaude Codeが登場します
このタグだらけのHTMLファイルを、あなたに代わって日本語の指示から直接書いてくれるのがClaude Code。
「申込みボタンの色を赤に変えて」と言えば、中身を書き換えて保存まで。
HTMLの書き方を覚える必要はありません。

Part 02

プラン変更のタイミング
― 課金しても待たされる理由 ―

第1回で実際に起きた事例：ワークショップ当日にトークンが尽きて、 Maxプランに課金したのに結局作業再開できずに1日が終わってしまった方が2名。
これを防ぐために、仕組みを理解しておきましょう。

2-1なぜ「Maxに上げても待たされる」のか

Claudeの利用制限は、Anthropic公式ヘルプによると「直近5時間のローリングウィンドウ方式」で判定されています。 つまり直近5時間の使用履歴が残り続ける仕組みです。

Maxへアップグレードしても即座に使えない場合があります
制限に到達した直後にMaxへアップグレードしても、直近5時間の使用量はそのまま残るため、 体感として「課金したのにすぐ使えない」というケースがあります（受講生からの報告ベース）。
※ 公式に「リセットされない」と明言されているわけではなく、ウィンドウが動くまでのタイムラグとして発生している現象です。
制限にひっかかる主な理由
  1. 5時間のローリングウィンドウ（公式）
    直近5時間の使用量がカウントされ続ける。一番多い原因。
  2. 週次制限（Weekly Cap）
    5時間ウィンドウとは別に「1週間単位」の上限もあり、Opus多用で届きやすい。
  3. モデル別の制限
    Opusは Sonnet より制限が厳しい。Maxでも Opus ばかりだと早く到達します。

2-2Pro と Max、どっちを選ぶ？判断の目安

月額3,000円と15,000円では差が大きいので、自分の使い方から判断するのが一番です。正解は一つではありません。

こんな使い方なら おすすめプラン

ChatGPTやNotebookLMと使い分けて節約
週1〜2回、数時間ずつ使う Pro で運用 → 足りなければMax検討
制限に毎日のようにひっかかる Max の方が快適
時間単価で考えると差額が回収できる
仕事で毎日Opusを使い倒したい Max 一択
制限を気にせず集中できる
ワークショップ当日のコツ
制限の仕組み上、当日にプランを上げても直近5時間の履歴は残るので、 すぐには快適にならない場合があります。Maxに切り替える場合は数日前に変えておくと安心。
当日制限が来たら、ChatGPT / NotebookLM で下準備を進めながら5時間待つのも有効な選択肢です。

Part 04

セキュリティ・連携

お客様の情報を扱うからこそ、気になるところ。
「連携」「許可」「個人情報」の3つのポイントを押さえましょう。

6-1「許可しますか？」と聞かれたとき

Claude Codeで作業中、「このファイルを変更していい？」と確認されることがあります。これは大事な安全機構です。

判断のポイント
  • 許可していい：自分のプロジェクトフォルダ内のファイル（HTML、images/内など）
  • ちょっと待って：見慣れないフォルダ、システム関連、消せないファイル
  • 断ってOK：意図しない場所を触ろうとしている、意味がわからない操作
わからない時の鉄則
「わかりません、どういう操作ですか？」と Claude Codeに聞いてみてください。
丁寧に説明してくれます。急いで「はい」と押さないで大丈夫。

6-2お客様の個人情報を扱うとき

基本ルール
お客様の氏名・電話番号・住所・決済情報などの個人情報は、
AIに直接貼り付ける前に匿名化するのが基本です。
安全に扱うための3ステップ
  1. 本当に必要か確認する
    LPを作るのに個人情報は本来不要。「お客様の声」として掲載する場合は事前に本人の許可を
  2. 匿名化する
    「Aさん・40代女性・東京都」のように情報を抽象化
  3. Drive連携は「特定のファイルのみ」に
    全体を見せず、渡したいファイルだけ明示的に選ぶ
参考：Anthropic（Claudeの会社）のポリシー
Claudeに入力した内容は、デフォルトではAIの学習には使われない設定になっています。 ただし100%安心ではないので、機密情報は一般論として扱うのがおすすめ。

2-1Windows：アプリを開こうとすると設定画面に飛ばされる

Windowsのみ

原因：Windowsが「Sモード」という制限付きモードになっていて、 Microsoft Store以外のアプリ（Claude Desktopなど）が入れられない状態です。
購入時から有効になっているPCがあるので、あなたのせいではありません。

確認方法
  1. 設定を開く
    スタート → 設定（歯車アイコン）
  2. システム → バージョン情報
    「Windowsの仕様」欄を確認

  • 翌日に開いたら画像データが消えていることがある
  • URL共有したときに文字化けする

Claude Codeでの管理 ― 3つのポイント

  1. 1つのLPに使うものは、1つのフォルダにまとめる

  2. 画像は「images/」という別フォルダにまとめる
    LPフォルダの中に images/ を作って、画像はそこに入れておく。こうしておくとClaudeがすぐに取りに行ってくれます。
  3. 名前は英語・パソコンが読みやすい単語で
    ターミナル上で操作する前提のツールなので、日本語ではなく英単語で。空白は -（ハイフン）か _（アンダースコア）で区切る。これを守らないと動作がおかしくなる原因になります。

Part 05

フォルダ整理の考え方

「images/ って何？」「新LPごとにフォルダ分けるべき？」 第1回で実際にあった質問への答えをまとめました。

3-1フォルダ＝引き出しのイメージ

パソコンのフォルダは、机の引き出しと同じ。
「何がどこにあるか」をあなた自身が説明できる状態にしておくのが目標です。

Claude Codeの視点
Claude Codeは、あなたが指定したフォルダの中だけが見えます。
だから、1つのLPに使う素材は1つのフォルダにまとめるのが基本。

3-2プロジェクト単位で分ける

お仕事ごと・LPごとに別フォルダ。これが基本ルールです。

推奨フォルダ構成の例
デスクトップ/
├── lp-aya-main/          ← 本命LP（あや）
│   ├── index.html
│   └── images/
│       ├── hero.jpg
│       └── profile.jpg
├── lp-aya-front/         ← フロントLP（あや）
│   ├── index.html
│   └── images/
└── lp-yumi-service/      ← ゆみさんサービスLP
    ├── index.html
    └── images/

3-3「images/」 って何？

images/ は「画像をまとめて入れておくフォルダ」という意味の慣習名です。
Claude Codeにも、HTMLから画像を読み込むときにも、この名前が使われます。

推奨

プロジェクトフォルダの中に images/ を作る

lp-front/
├── index.html
└── images/ ← ここに画像

混乱しがち

デスクトップに直接「写真素材」フォルダを作って、そこに入れる

LPが画像を探せず表示されないことがある

ルール
HTMLファイルと同じフォルダの中に、images/ を作る。
これだけ守れば迷いません。

3-4新しいLPを作るとき、フォルダは別にする？

はい、別フォルダにするのが基本です。

  • 本命LPとフロントLPは別フォルダに
  • クライアントAの案件とクライアントBの案件は別フォルダに
  • 理由：素材が混ざらない・Claude Codeが混乱しない・あとで見返しやすい
新しいLPを始めるときの型
  1. デスクトップ（か専用の場所）に新フォルダを作る
    例：lp-yoshimi-front/
  2. その中に images/ フォルダを作る
  3. 使いたい画像をimages/に入れる
    ファイル名は英語で、空白なしで
  4. 提案書・原稿があれば同じフォルダの中に
    例：brief.md や copy.txt
  5. Claude Codeでそのフォルダを開く

3-5日付の入れ方

日付をフォルダ名やファイル名に入れるときは、並び替えやすい形式で。

Good
  • 20260423-qa-session/
  • 2026-04-23_meeting.md
  • 202604_monthly-report/

並び替えると日付順に揃う

Bad
  • 4月23日_打合せ/
  • 23-4-2026.md
  • 最新版_ver2_final.md

並び替えがバラバラになる

3-6当日までに用意しておくと差がつくもの

第1回の受講生から「これがあれば当日もっと進んだ」と聞いた項目です。

事前に揃えておく7つのもの
  • 使いたい写真を10〜20枚（プロフィール・サービス風景・雰囲気写真）
  • 参考にしたいLPのURL 3〜5本（雰囲気のモデリング先）
  • キャッチコピーの候補（10個くらい書き出しておく）
  • サービス説明文（長くてOK、あとで整える）
  • お客様の声（あれば2〜3件）
  • ロゴ（PNGの透過版があるとベスト）
  • ブランドカラー（3色くらい決めておく）
これらを入れるフォルダ
1つのプロジェクトフォルダにまとめて、その中で
images/、copy/（文章）、brief/（提案書）などに整理しておくとClaude Codeがスムーズに読めます。

3-7ファイル名を一括で英語にしてもらうコツ

日本語で「スクリーンショット 2026-04-22 14.32.10.png」みたいなファイル名、そのままだとLPで使えません。
でも、自分で1枚ずつ名前を変えるのは大変…

朗報
images/ フォルダに全部ぶち込んで、Claude Codeに一括リネームしてもらえばOK。
中身を見て、用途に合った英語名を考えて、ぜんぶ一気に変えてくれます。
やり方の流れ
  1. images/ に画像を全部入れる
    日本語名・スマホ撮影そのままでOK。とりあえず入れる
  2. Claude Codeに頼む
    例：「images/の中のファイル名を、中身を見て英語のわかりやすい名前に変えて」
  3. 提案を確認する
    Claudeが「hero.jpg / profile-smile.jpg / service-scene-01.jpg …」のように案を出すので、OKを出す
  4. 一括でリネーム
    許可を出すと、一気に変えてくれます
そのまま使える頼み方テンプレ
images/ の画像を見て、
中身がわかる英語のファイル名に
一括で変更してください。

ルール：
・小文字 + ハイフン区切り（例：hero-image.jpg）
・拡張子はそのまま
・同じ種類が複数あるなら 01, 02 と連番
・変更前後の対応表も見せて
一手間かけると安心
心配なら、リネーム前のフォルダをコピーしておくと元に戻せます。
「images-backup/」みたいな名前で保管しておくのがおすすめ。
なぜ英語名がいいの？
日本語ファイル名は、URLに含まれると文字化けしたり、ブラウザによって画像が表示されないことがあります。
最初から英語で揃えておけば、公開してから「画像が出ない！」というトラブルを防げます。
Part 06

画像・ファイル名のコツ

地味だけど、ここでつまずくとLPが表示されなかったり、文字化けしたり…。 最初に「型」を身につけておくと、あとがラクです。

4-1画像フォーマットの使い分け（PNG / JPG / WebP）

JPG
写真向き・軽い
お客様写真・風景・FV背景に。
透過はできない
PNG
透過・文字入り向き
ロゴ・スクショ・アイコン。
写真に使うと重い
WebP
軽さと透過の両立
サイト高速化の切り札。
一部古いブラウザで×
サイズの目安
LP1ページに使う画像は、1枚あたり500KB以下を目標に。
3MBの写真をそのまま入れると、スマホで開くのに5秒かかることも。

4-2ファイル名は「英語」で書く

なぜ？ 日本語ファイル名はWeb上で文字化けしたり、URLが壊れたりするから。
英語なら世界中のどのサーバーでも確実に動きます。

Good
  • hero-image.jpg
  • logo.png
  • customer-voice-01.jpg
  • about-photo.webp
Bad
  • トップ画像.jpg
  • ロゴ.png
  • お客様の声①.jpg
  • プロフィール写真.webp

4-3空白は入れない。代わりにハイフン（-）

ファイル名に半角スペース・全角スペースを入れると、URLが途中で切れたり、正しく読み込めないことがあります。

Good
  • hero-image.jpg（ハイフン）
  • hero_image.jpg（アンダースコア）
  • heroImage.jpg（大文字区切り）
Bad
  • hero image.jpg（空白あり）
  • hero　image.jpg（全角空白）
  • hero(1).jpg（カッコは避ける）
迷ったら
全部小文字・ハイフン区切りが一番安全。
例：aya-profile-01.jpg / service-menu-top.png
