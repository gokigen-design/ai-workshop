Part 01

まず知っておくこと
― Claudeの全体像 ―

「なぜこの操作が必要なの？」がわかると、トラブルが起きたときも自分で判断できます。 まずは全体の地図を頭に入れましょう。

1-1Claudeは「3種類」あります

みなさんが「Claude」と呼んでいるものは、実は3つの別物です。それぞれ得意なことが違います。

種類 何ができる？ どんな時に使う？
Claude（ブラウザ）
claude.ai		 文章を書く・要約・アイデア出し・画像の説明 ちょっと調べたい時・壁打ち
Claude Desktop
アプリ版		 ブラウザ版とほぼ同じ。スマホアプリもここに近い PCにアプリとして常駐させたい時
Claude Code
本命		 PCのファイルを直接さわれる・LPを作れる・フォルダを読める ワークショップで使うのはコレ
大事なのはココ
チャットのClaudeとClaude Codeは別アプリだけど、同じアカウントで動いている。 そしてProプランの場合、トークン（使える量）は共通の財布から減っていきます。 詳しくは 1-3 で説明します。

1-3プラン・課金・制限の仕組み

第1回で一番つまずいた「気づいたら制限がきた」問題。ここを丁寧に解説します。

プラン 料金（月） 使えるモデル 向いている人
Free 0円 Sonnet（制限あり） お試しだけ
Pro 約3,000円
($20)		 Sonnet / Opus
（量に上限あり）		 ワークショップ参加者の基本
Max 約15,000円
($100)〜		 Opus常用OK
（制限ほぼ気にならない）		 毎日がっつり使う人
超重要：チャットとCodeは「同じ財布」です
Proプランでは、チャットで使ったトークンも Claude Codeで使ったトークンも、 同じ上限からカウントされます。

「チャットでたくさん壁打ち」→「Codeで作業しようとしたら制限」というパターンに注意。
トークンを節約する3つのコツ

① 使い分け戦略（他の無料AIと組み合わせる）

目的 おすすめツール 理由
壁打ち・アイデア出し ChatGPT（無料） 雑談や発想はこっちで消費

最終仕上げ・コード Claude Code トークンを温存して本番で使う

② モデル切り替え

  • 普段の作業は Sonnet（早い・トークン消費少ない）
  • なかなか良い回答が出ないとき → Opusに切り替える（精度優先）

③ Maxプランという選択肢

「節約するより、気にせず使いたい」なら Max へ。常に Opus が使えて、

制限がきた時の対処法

  • 別のAIで下準備：ChatGPTやNotebookLMで続きを
  • プランを上げる：毎日制限がくるならMaxへ

Part 02

セットアップで
つまずくポイント

第1回でみなさんが実際にハマったところを、原因と対処をセットで全部書いておきます。 OSごとに分けているので、自分のPCに合わせて読んでください。

2-1Windows：アプリを開こうとすると設定画面に飛ばされる

Windowsのみ

原因：Windowsが「Sモード」という制限付きモードになっていて、 Microsoft Store以外のアプリ（Claude Desktopなど）が入れられない状態です。
購入時から有効になっているPCがあるので、あなたのせいではありません。

確認方法
  1. 設定を開く
    スタート → 設定（歯車アイコン）
  2. システム → バージョン情報
    「Windowsの仕様」欄を確認
  3. 「Sモード」と書かれているか？
    書かれていたらSモードです
Sモードを解除する手順（無料）
  1. 設定 → システム → ライセンス認証
  2. 「Microsoft Storeに移動」をクリック
  3. 「Sモードをオフにする」ページで「入手」
  4. PCを再起動
  5. ダウンロード済みのセットアップファイルを実行
安心ポイント
解除は無料です。一度解除すると元には戻せませんが、
「特別な保護を外す」というよりは「普通のWindowsに戻す」イメージ。 ほとんどのWindows PCはこのモードなしで使われています。
Sモードじゃなかったら
セットアップファイルを右クリック → 「管理者として実行」を試してください。

2-2「デバイスに変更を加えることを許可しますか？」

Windowsのみ

インストール中にこの確認が出たら、「はい」で大丈夫です。これは「このアプリを入れていいですか？」という普通の確認です。

ただし
このメッセージは信頼できるアプリを入れるときだけ「はい」にしてください。 知らないアプリで出たら一度立ち止まって確認を。

2-3Mac：Gitを入れようとして詰まった

Macのみ

結論：Macは Git を自分で入れる必要はありません
Macには最初から Git が入っています。
Windowsユーザー向けの手順を見て混乱しないよう、STEP01のMac手順だけ見てください。

Xcode Command Line Tools を入れようとして「macOSのバージョンが新しすぎる」と出た場合も、 無視してOKです。Claude CodeをMacで使うのに必須ではありません。

参考：事前準備ガイド（preparation.html）のMac手順

2-4ブラウザ拡張機能が入らない

Win / Mac

ChromeやEdgeの拡張機能（GoFullPageなど）が入らない場合のチェックポイント：

  • ブラウザが最新バージョンか確認
  • シークレットモードでは拡張は入らないことがある → 通常モードで
  • 会社や学校のPCは管理者が制限していることがある
  • Chromeウェブストアから入れようとしているか（他サイトからは危険）

2-5スマホはProなのにPCでログインできない

「スマホでProに課金しているのに、PCでログインしたらアカウント作成画面が出る」 というケース。これはログイン方法が違う可能性が高いです。

解決ステップ
  1. スマホのClaudeアプリを開く
  2. 設定 or アカウント情報を確認
    登録に使っているメールアドレスをメモ
  3. PCでは同じ方法でログイン
    Google / Apple / メール のうち同じもの
特に注意：Appleでサインイン
iPhoneで「Appleでサインイン」を使っていた場合、
@privaterelay.appleid.com みたいな非公開メールになっていることがあります。
その場合はPCでも必ず「Appleでサインイン」を選んでください。

同じアカウントでログインできれば、すでにProなので追加課金は不要です。

Part 03

画像・ファイル名のコツ

地味だけど、ここでつまずくとLPが表示されなかったり、文字化けしたり…。 最初に「型」を身につけておくと、あとがラクです。

3-1ファイル名は「英語」で書く

なぜ？ 日本語ファイル名はWeb上で文字化けしたり、URLが壊れたりするから。
英語なら世界中のどのサーバーでも確実に動きます。

Good
  • hero-image.jpg
  • logo.png
  • customer-voice-01.jpg
  • about-photo.webp
Bad
  • トップ画像.jpg
  • ロゴ.png
  • お客様の声①.jpg
  • プロフィール写真.webp

3-2空白は入れない。代わりにハイフン（-）

ファイル名に半角スペース・全角スペースを入れると、URLが途中で切れたり、正しく読み込めないことがあります。

Good
  • hero-image.jpg（ハイフン）
  • hero_image.jpg（アンダースコア）
  • heroImage.jpg（大文字区切り）
Bad
  • hero image.jpg（空白あり）
  • hero　image.jpg（全角空白）
  • hero(1).jpg（カッコは避ける）
迷ったら
全部小文字・ハイフン区切りが一番安全。
例：aya-profile-01.jpg / service-menu-top.png

3-3画像フォーマットの使い分け（PNG / JPG / WebP）

形式 得意 苦手 こんな時に
JPG
.jpg / .jpeg		 写真・グラデーション。軽い 透過できない お客様写真・風景・FV背景
PNG
.png		 文字入り画像・ロゴ・透過 写真に使うと重い ロゴ・スクショ・アイコン
WebP
.webp		 JPGより軽くて、透過もOK 一部古いブラウザで見えない サイトを軽くしたい時の切り札
サイズの目安
LP1ページに使う画像は、1枚あたり500KB以下を目標に。
3MBの写真をそのまま入れると、スマホで開くのに5秒かかることも。

3-4スクリーンショットの撮り方

Mac

Command + Shift + 4
→ ドラッグして範囲選択で保存

Command + Shift + 5
→ 細かいオプション（画面録画も）

Windows

Windows + Shift + S
→ Snipping Toolが起動

Print Screen
→ 画面全体を撮影

撮ったらすぐ名前を変える
スクリーンショット 2026-04-18 13.24.51.png のまま置いておくと、 あとで「どれが何のスクショだっけ…」となります。 撮った直後に英語の意味のある名前に変えるクセを。

3-5LPの参考にしたい時の拡張ツール

他の素敵なLPを見つけたとき、全体を1枚の画像として保存できるツールがあります。

GoFullPage（Chrome拡張）

Chromeウェブストアで無料で入れられます。
ボタン1つで、縦に長いLP全体をスクショできます。
「この参考LPの全体像を残したい」というときに便利。

3-6素材はどこから集める？

「いい素材がない…」がLP制作の一番の詰まりポイント。日頃から集めておくと差がつきます。

ソース 特徴 URL / 場所
Unsplash 海外写真の王道・商用OK・無料 unsplash.com
Pexels 写真＆動画どちらも無料 pexels.com
O-DAN 日本語で海外サイトを横断検索 o-dan.net
Canva Pro素材 課金者は数百万点の素材が使える Canva内で直接検索
AI画像生成 ブランドに合わせて生成できる Midjourney / ChatGPT画像
自分で撮影 「その人らしさ」が一番出る スマホ＋自然光で十分
プロの習慣
「素敵だな」と思ったらその場でスクショ・URLを保存。
専用フォルダ（例：inspiration/）を作って、
気に入ったLP・写真・配色を片っ端から入れておくと、いざという時の引き出しになります。
Part 04

フォルダ整理の考え方

「images/ って何？」「新LPごとにフォルダ分けるべき？」 第1回で実際にあった質問への答えをまとめました。

4-1フォルダ＝引き出しのイメージ

パソコンのフォルダは、机の引き出しと同じ。
「何がどこにあるか」をあなた自身が説明できる状態にしておくのが目標です。

Claude Codeの視点
Claude Codeは、あなたが指定したフォルダの中だけが見えます。
だから、1つのLPに使う素材は1つのフォルダにまとめるのが基本。

4-2プロジェクト単位で分ける

お仕事ごと・LPごとに別フォルダ。これが基本ルールです。

推奨フォルダ構成の例
デスクトップ/
├── lp-aya-main/          ← 本命LP（あや）
│   ├── index.html
│   └── images/
│       ├── hero.jpg
│       └── profile.jpg
├── lp-aya-front/         ← フロントLP（あや）
│   ├── index.html
│   └── images/
└── lp-yumi-service/      ← ゆみさんサービスLP
    ├── index.html
    └── images/

4-3「images/」 って何？

images/ は「画像をまとめて入れておくフォルダ」という意味の慣習名です。
Claude Codeにも、HTMLから画像を読み込むときにも、この名前が使われます。

推奨

プロジェクトフォルダの中に images/ を作る

lp-front/
├── index.html
└── images/ ← ここに画像

混乱しがち

デスクトップに直接「写真素材」フォルダを作って、そこに入れる

LPが画像を探せず表示されないことがある

ルール
HTMLファイルと同じフォルダの中に、images/ を作る。
これだけ守れば迷いません。

4-4新しいLPを作るとき、フォルダは別にする？

はい、別フォルダにするのが基本です。

  • 本命LPとフロントLPは別フォルダに
  • クライアントAの案件とクライアントBの案件は別フォルダに
  • 理由：素材が混ざらない・Claude Codeが混乱しない・あとで見返しやすい
新しいLPを始めるときの型
  1. デスクトップ（か専用の場所）に新フォルダを作る
    例：lp-yoshimi-front/
  2. その中に images/ フォルダを作る
  3. 使いたい画像をimages/に入れる
    ファイル名は英語で、空白なしで
  4. 提案書・原稿があれば同じフォルダの中に
    例：brief.md や copy.txt
  5. Claude Codeでそのフォルダを開く

4-5日付の入れ方

日付をフォルダ名やファイル名に入れるときは、並び替えやすい形式で。

Good
  • 20260423-qa-session/
  • 2026-04-23_meeting.md
  • 202604_monthly-report/

並び替えると日付順に揃う

Bad
  • 4月23日_打合せ/
  • 23-4-2026.md
  • 最新版_ver2_final.md

並び替えがバラバラになる

4-6当日までに用意しておくと差がつくもの

第1回の受講生から「これがあれば当日もっと進んだ」と聞いた項目です。

事前に揃えておく7つのもの
  • 使いたい写真を10〜20枚（プロフィール・サービス風景・雰囲気写真）
  • 参考にしたいLPのURL 3〜5本（雰囲気のモデリング先）
  • キャッチコピーの候補（10個くらい書き出しておく）
  • サービス説明文（長くてOK、あとで整える）
  • お客様の声（あれば2〜3件）
  • ロゴ（PNGの透過版があるとベスト）
  • ブランドカラー（3色くらい決めておく）
これらを入れるフォルダ
1つのプロジェクトフォルダにまとめて、その中で
images/、copy/（文章）、brief/（提案書）などに整理しておくとClaude Codeがスムーズに読めます。
Part 05

Claude Codeでできること

できること / できないこと / チャットとの使い分け。 「何を頼めばいいか」の全体像がわかると、活用の幅が一気に広がります。

5-1できること一覧

  • LP・HPの制作（HTML / CSS）
  • 既存サイトの修正（色・文字・レイアウト変更）
  • ブログ・会員サイトの作成
  • LINE/メール配信の自動化スクリプト作成
  • スプレッドシート連携（Google Apps Script）
  • フォルダ内の書類を横断的に読んで要約
  • 大量ファイルの一括リネーム・整理
  • PDF・画像からの文字起こし
  • GitHubへのアップ・Vercelデプロイ

5-2できないこと（苦手なこと）

  • 画像そのものを作る（生成はChatGPTやMidjourneyで）
  • 動画編集（Canva・CapCutなど専門ツールへ）
  • 有料ツールの自動ログイン操作
  • リアルタイムな株価・天気など最新情報の取得（Web検索は限定的）
  • あなたの確認なしに重要な削除

5-3チャット × Claude Code の使い分け

受講生から多かった質問：「チャットで話した内容はCodeに引き継がれる？」

答え：引き継がれません
チャットとCodeは別アプリです。チャットでの会話はCode側には見えません。
大事な設定や人物像は、ファイルに書いて保存しておく必要があります。
やりたいこと 使うべきは
アイデア出し・雑談・壁打ち チャット（またはChatGPTで節約）
文章を整える チャット
画像を見せて意見を聞く チャット
LP・HPを作る Claude Code
既存ファイルを編集する Claude Code
フォルダの中身を理解させる Claude Code
事前課題のとき、どっちで壁打ちする？

受講生のよくある迷い：「ステップ2の内容整理は、Codeと先に壁打ちしてから埋める？ 埋めてから壁打ちする？」

おすすめの順番：

  1. まずはチャット（またはChatGPT）で壁打ち
    アイデアがぐちゃぐちゃな段階はここで整理（トークン節約）
  2. ある程度まとまったらステップ2を書く
    完璧じゃなくてOK
  3. Claude Codeで実際にLP化しながら調整
    ここでトークンをしっかり使う
Part 06

セキュリティ・連携

お客様の情報を扱うからこそ、気になるところ。
「連携」「許可」「個人情報」の3つのポイントを押さえましょう。

6-1Google Drive連携：3つの選択肢

Drive連携で出てくる選択肢、迷いますよね。あなたのやりたいこと別にまとめました。

あなたの使い方 選ぶもの 注意点
Driveの大量資料をまとめて読ませたい ①すべてのファイルの表示・DL Drive全体が見える状態に。機密ファイルがあれば注意
特定ファイルだけ読ませたい ②特定のファイルのみ 一番安全。使うときに都度選ぶ
Driveとは連携せずローカルで作業 どちらもチェックせずスキップ Claude CodeでLPを作るだけなら、これで十分
迷ったら
まずは「チェックせずスキップ」でOK。 あとからGoogleアカウントの設定でいつでも変更できます。

6-2「許可しますか？」と聞かれたとき

Claude Codeで作業中、「このファイルを変更していい？」と確認されることがあります。これは大事な安全機構です。

判断のポイント
  • 許可していい：自分のプロジェクトフォルダ内のファイル（HTML、images/内など）
  • ちょっと待って：見慣れないフォルダ、システム関連、消せないファイル
  • 断ってOK：意図しない場所を触ろうとしている、意味がわからない操作
わからない時の鉄則
「わかりません、どういう操作ですか？」と Claude Codeに聞いてみてください。
丁寧に説明してくれます。急いで「はい」と押さないで大丈夫。

6-3お客様の個人情報を扱うとき

基本ルール
お客様の氏名・電話番号・住所・決済情報などの個人情報は、
AIに直接貼り付ける前に匿名化するのが基本です。
安全に扱うための3ステップ
  1. 本当に必要か確認する
    LPを作るのに個人情報は本来不要。「お客様の声」として掲載する場合は事前に本人の許可を
  2. 匿名化する
    「Aさん・40代女性・東京都」のように情報を抽象化
  3. Drive連携は「特定のファイルのみ」に
    全体を見せず、渡したいファイルだけ明示的に選ぶ
参考：Anthropic（Claudeの会社）のポリシー
Claudeに入力した内容は、デフォルトではAIの学習には使われない設定になっています。 ただし100%安心ではないので、機密情報は一般論として扱うのがおすすめ。
