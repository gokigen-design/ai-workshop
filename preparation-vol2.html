心構え
プロのデザイナーはLPをこう見る

CCでLPを作る技術より大事なこと。それが「デザイナーの頭の中」です。当日の前に読んでおいてください。

「準備が9割」
CCで作れることは手段。
何を伝えるかが、すべてです。

どんなに綺麗なLPでも、伝えるべき情報が整っていなければ申込にはつながりません。当日CCに指示する前に、「何を・誰に・どう伝えるか」を決めておくことが最大の準備です。

1
アートとデザインは違う
Art / アート
自己表現。作者が伝えたいことを形にする。「好き」「美しい」が基準。見る人の解釈に委ねる。
Design / デザイン
目的達成のための設計。見る人に「行動してもらう」ことが基準。綺麗かどうかは手段でしかない。

LPはデザインです。「おしゃれに見せたい」は手段。「申し込んでもらう」「問い合わせしてもらう」が目的です。

2
情報をデザインするとは

LPに載せる情報は「全部」ではなく「必要なものだけ」。情報の取捨選択こそがデザインの第一歩です。

1
このLPで何をしてほしいか、1つ決める
「申し込んでほしい」「問い合わせしてほしい」「LINEに登録してほしい」。ゴールは必ず1つ。複数あると何もしてもらえません。
2
誰に届けたいかを1人に絞る
「全員に刺さるLP」は誰にも刺さりません。理想の1人を思い浮かべて書くと、読んだ人が「これ私のことだ」と感じます。
3
載せない情報を決める勇気を持つ
「全部説明しなきゃ」という気持ちはわかります。でも情報が多いほど読まれません。情報の取捨選択、優先順位を考えるのがデザインです。
3
だから準備が9割

CCはあなたの指示通りに作ります。指示の質＝LPの質です。

準備が足りないとき
「なんか違う」「イメージと違う」を繰り返す
→
準備できているとき
1回の指示で8割完成。あとは細かい調整だけ

ワークシートにしっかり記入することで、CCへの指示の質が上がります。タブ07「ワークシート」を丁寧に埋めてから、当日に臨んでください。

4
その上で、デザインを選ぶ

伝える内容が決まってはじめて、デザインの出番です。カラー・フォント・素材は「誰に届けたいか」から選びます。

Color
色はターゲットの「好みの世界観」から選ぶ。自分の好みではなく、お客さんが好きな色。
Font

Claude Code 機能を使ってみよう

Claude Desktop には「Claude Code」機能が内蔵されています。普通のチャットに加えて、あなたのPC上でファイルを作成したり、プログラムを実行したりできる機能です。ターミナルを開く必要はありません。

Claude Code 機能でできること
ファイル作成 — 企画書・レポート・HTML・スプレッドシートなどを PC 上に直接保存
データ処理 — Excel / CSV ファイルを読み込んで変換・分析・整形
ツール構築 — あなたの仕事を自動化するスクリプトを作成して実行
Webサイト作成 — LP やポートフォリオをその場で構築
普通のチャットと何が違う？
普通のチャット → 回答をコピーして自分で貼り付ける。
Claude Code 機能 → Claude が直接あなたの PC にファイルを作る。コピペ不要。「LP作って」と言えば HTML ファイルがデスクトップに出来上がる。

─ 環境構築でつまずいた時
Mac で「xcode-select --install」というエラーが出ました

Mac に必要な開発ツール（Xcode Command Line Tools）が入っていないだけです。

1. ⌘ + space →「ターミナル」と入力して開く
2. xcode-select --install をコピペして Enter
3. ポップアップで「インストール」をクリック → 完了したら Claude Code を再起動

解決しない時はエラー画面をスクショ（⌘ + shift + 4）して Claude のチャットに貼り付けて聞いてください。

Windows で「Git をインストールしてください」と出ました

Claude Code は内部で Git というソフトを使うので、先に Git for Windows をインストールする必要があります。

1. git-scm.com/downloads/win から「Click here to download」をクリック
2. ダウンロードしたファイルを開いて、すべて「Next」を押し続けて Install → Finish
3. Claude Code を再起動

Google アカウントでログインできない

メールアドレスで新規アカウントを作成してください。Claude のアカウントは Google ログインではなくメール登録が確実です。

Desktop アプリが開かない / Windows でブロックされる

PC を再起動してからもう一度試してください。Windows で「WindowsによってPCが保護されました」と出たら、「詳細情報」→「実行」をクリックすれば進めます。

─ Claude Code の使い方
「フォルダ」って何？どこに作ればいい？

Claude Code は「指定したフォルダの中だけ」を見るしくみです。PC全体を見られないように、作業場所を1つ決めてあげるのがフォルダ選択です。

おすすめ：デスクトップに「クラウドコード」というフォルダを作り、その中に案件ごとのサブフォルダを作る。
例：クラウドコード / LP制作 / 5月9日ワークショップ

素材ファイル（写真・参考資料など）も、Claude に見せたければそのフォルダにコピーして入れておきましょう（移動ではなくコピー）。

「許可を求められた」のダイアログ、Yes押して大丈夫？

自分の作業フォルダの中なら全部 Yes でOKです。Claude は安全のため、ファイルを書き換える前に毎回確認してきます。慣れてきたら「このセッションの間は許可」を選ぶと聞かれなくなります。

不安な時はそのまま読んで判断すれば大丈夫。怪しい操作（パスワードや決済まわり）でない限り、Yesで進めて問題ありません。

Pro と Max、どっちのプランがいい？

ワークショップは Pro（$22／月・税込）でOKです。Pro で十分LPは作れます。

Max（$100）は1日に何時間も Claude Code を使い倒したい人向け。回数制限に当たって困るようになったらアップグレードを検討してください（いつでも変更可）。

「Sonnet」「Opus」ってモデル選択、どっちにすればいい？

基本は Sonnet で大丈夫です。動作も速くて、LP制作にはこれで十分。

「もっとデザインにこだわりたい」「複雑な指示を理解させたい」時だけ Opus に切り替えてみてください。Opus は賢い分、消費が大きいので使い分けがおすすめ。

既存ファイルや素材を、新しいフォルダに「移動」していい？

移動ではなく「コピー」してください。元のファイルはそのままに、コピーをClaude用フォルダに入れる。これで「あ、消えちゃった！」を防げます。

スマホからでもできる？

パソコンを強くおすすめします。Claude Desktop アプリも Claude Code 機能もパソコン用です。スマホでは Claude のチャットアプリは使えますが、ファイル作成などの本格的な作業はできません。

デザインの4つのルール

これだけ押さえれば「ダサい」は回避できます。

余白を恐れない
詰め込みすぎは読みにくさの原因。空白は「洗練」のサイン。要素間は広めにとる。
色は3色まで
メイン・サブ・アクセントの3色に絞る。多色使いはアマチュア感が出てしまう。
写真は明るく・鮮明に
暗い・ぼやけた写真は信頼感を下げる。顔が見える・明るい写真が最強の素材。
テキストは短く・端的に
LPは「読まれない」前提で作る。一文30文字以内を目安に。キャッチコピーはさらに短く。

日本語でどう伝えればいい？

CCは普通の日本語で話しかけるだけでOKです。難しい言葉は不要。ポイントは「何を・どこに・どう変えたいか」を一文で伝えること。

1
場所を具体的に言う
「直して」だけではなく、「一番上の大きい文字を」「ボタンの色を」など、どこかを先に言う。
2
変えたいことと変えたくないことを両方言う
「色だけ変えて、文章はそのままで」のように、触ってほしくない部分も伝えると安心。
3
気分・雰囲気で伝えてもOK
「もっとやわらかい雰囲気に」「高級感を出して」「ちょっと余白増やして」でちゃんと伝わります。

自分の作業フォルダの中なら全部 Yes でOKです。Claude は安全のため、ファイルを書き換える前に毎回確認してきます。慣れてきたら「このセッションの間は許可」を選ぶと聞かれなくなります。

不安な時はそのまま読んで判断すれば大丈夫。怪しい操作（パスワードや決済まわり）でない限り、Yesで進めて問題ありません。

Pro と Max、どっちのプランがいい？

ワークショップは Pro（$22／月・税込）でOKです。Pro で十分LPは作れます。

Max（$100）は1日に何時間も Claude Code を使い倒したい人向け。回数制限に当たって困るようになったらアップグレードを検討してください（いつでも変更可）。

2-1なぜ「Maxに上げても待たされる」のか

Claudeの利用制限は、Anthropic公式ヘルプによると「直近5時間のローリングウィンドウ方式」で判定されています。 つまり直近5時間の使用履歴が残り続ける仕組みです。

Maxへアップグレードしても即座に使えない場合があります
制限に到達した直後にMaxへアップグレードしても、直近5時間の使用量はそのまま残るため、 体感として「課金したのにすぐ使えない」というケースがあります（受講生からの報告ベース）。
※ 公式に「リセットされない」と明言されているわけではなく、ウィンドウが動くまでのタイムラグとして発生している現象です。
制限にひっかかる主な理由
  1. 5時間のローリングウィンドウ（公式）
    直近5時間の使用量がカウントされ続ける。一番多い原因。
  2. 週次制限（Weekly Cap）
    5時間ウィンドウとは別に「1週間単位」の上限もあり、Opus多用で届きやすい。
  3. モデル別の制限
    Opusは Sonnet より制限が厳しい。Maxでも Opus ばかりだと早く到達します。

6-1「許可しますか？」と聞かれたとき

Claude Codeで作業中、「このファイルを変更していい？」と確認されることがあります。これは大事な安全機構です。

判断のポイント
  • 許可していい：自分のプロジェクトフォルダ内のファイル（HTML、images/内など）
  • ちょっと待って：見慣れないフォルダ、システム関連、消せないファイル
  • 断ってOK：意図しない場所を触ろうとしている、意味がわからない操作
わからない時の鉄則
「わかりません、どういう操作ですか？」と Claude Codeに聞いてみてください。
丁寧に説明してくれます。急いで「はい」と押さないで大丈夫。

  2. 匿名化する

AIっぽさを見抜く
13-A ─ AIっぽいデザインの正体

「Claudeに頼んだら、なんか既視感のあるLPが出てきた…」その違和感の正体は、実はたった6つの特徴に集約できます。これを知っているだけで、出来上がったLPの"AIっぽさ"を自分でチェックできるようになります。

よくある6つの特徴
この6つに当てはまったら「AIっぽい」サイン

ざっくり指示すると、Claudeはほぼ毎回この組み合わせで返してきます。パッと見「悪くないけど、なんか普通…」と感じたら、たいていここに該当しています。

01
角がぜんぶ丸い
カード・ボタン・画像、すべての角が同じくらいの丸み。柔らかく見えるけど、印象がのっぺりして"優しすぎる"見た目に。
02
絶妙なベージュ・パステル
一見オシャレに見える、薄いベージュやパステル系の絶妙な配色。色に強弱がなく、印象に残らないトーンに収まる。
03
3カラム＋丸アイコン
「特徴3つ」を3カラムで並べ、それぞれに丸い線アイコンが乗る構成。テンプレ感がもっとも出やすい組み合わせ。
04
紫→ピンクのグラデ
FVや見出しに敷かれる、紫からピンクへの斜めグラデ。「AIに作らせました」と言わんばかりの定番カラー。
05
中央寄せ・等間隔
どのセクションも同じ余白で中央寄せ。リズムや強弱がないため、スクロールしても記憶に残らない。
06

海外モデルの笑顔・握手の写真がそのまま使われている。サービスの空気感とちぐはぐで、距離を感じさせる。

13-C ─ 動きで魅せる（アニメーション）

5つの定番アニメーションを使い分け
