Mindset
プロのデザイナーはLPをこう見る

CCでLPを作る技術より大事なこと。それが「デザイナーの頭の中」です。当日の前に読んでおいてください。

「準備が9割」
CCで作れることは手段。
何を伝えるかが、すべてです。

どんなに綺麗なLPでも、伝えるべき情報が整っていなければ申込にはつながりません。当日CCに指示する前に、「何を・誰に・どう伝えるか」を決めておくことが最大の準備です。

1
アートとデザインは違う
Art / アート
自己表現。作者が伝えたいことを形にする。「好き」「美しい」が基準。見る人の解釈に委ねる。
Design / デザイン
目的達成のための設計。見る人に「行動してもらう」ことが基準。綺麗かどうかは手段でしかない。

LPはデザインです。「おしゃれに見せたい」は手段。「申し込んでもらう」「問い合わせしてもらう」が目的です。

2
情報をデザインするとは

LPに載せる情報は「全部」ではなく「必要なものだけ」。情報の取捨選択こそがデザインの第一歩です。

1
このLPで何をしてほしいか、1つ決める
「申し込んでほしい」「問い合わせしてほしい」「LINEに登録してほしい」。ゴールは必ず1つ。複数あると何もしてもらえません。
2
誰に届けたいかを1人に絞る
「全員に刺さるLP」は誰にも刺さりません。理想の1人を思い浮かべて書くと、読んだ人が「これ私のことだ」と感じます。
3
載せない情報を決める勇気を持つ
「全部説明しなきゃ」という気持ちはわかります。でも情報が多いほど読まれません。情報の取捨選択、優先順位を考えるのがデザインです。
3
だから準備が9割

CCはあなたの指示通りに作ります。指示の質＝LPの質です。

準備が足りないとき
「なんか違う」「イメージと違う」を繰り返す
→
準備できているとき
1回の指示で8割完成。あとは細かい調整だけ

ワークシートにしっかり記入することで、CCへの指示の質が上がります。タブ07「ワークシート」を丁寧に埋めてから、当日に臨んでください。

4
その上で、デザインを選ぶ

伝える内容が決まってはじめて、デザインの出番です。カラー・フォント・素材は「誰に届けたいか」から選びます。

Color
色はターゲットの「好みの世界観」から選ぶ。自分の好みではなく、お客さんが好きな色。
Font
フォントはサービスの「声のトーン」。高単価なら知的に。ママ向けならやわらかく。
Photo
素材はLPの印象を9割決める。暗い写真・ぼやけた写真は信頼を下げる。

3
Claude Code 機能を使ってみよう

Claude Desktop には「Claude Code」機能が内蔵されています。普通のチャットに加えて、あなたのPC上でファイルを作成したり、プログラムを実行したりできる機能です。ターミナルを開く必要はありません。

Claude Code 機能でできること
ファイル作成 — 企画書・レポート・HTML・スプレッドシートなどを PC 上に直接保存
データ処理 — Excel / CSV ファイルを読み込んで変換・分析・整形
ツール構築 — あなたの仕事を自動化するスクリプトを作成して実行
Webサイト作成 — LP やポートフォリオをその場で構築
普通のチャットと何が違う？
普通のチャット → 回答をコピーして自分で貼り付ける。
Claude Code 機能 → Claude が直接あなたの PC にファイルを作る。コピペ不要。「LP作って」と言えば HTML ファイルがデスクトップに出来上がる。
まずは基本のチャットから練習してみましょう。
1
まずは話しかけてみる
「私はオンライン秘書をしています。クライアントとの打ち合わせ内容を箇条書きでメモしたのですが、これを綺麗な議事録に整えてもらえますか？」
→ Claude が議事録のフォーマットを提案してくれます。メモを貼り付けると整形してくれます。
2
ファイルを渡してみる
チャット欄の左下にあるクリップマークをクリックして、手元のファイル（Excel・PDF・画像など）を添付。「このデータを分析して」「この資料を要約して」と伝えてみてください。
→ Claude がファイルの中身を読み取って回答してくれます。
3
Claude Code 機能を試す
「Claude Code 機能を使って、LINE配信用の告知文を3パターン作って、テキストファイルとしてデスクトップに保存してください。講座名は「はじめての米粉スイーツ講座」、日時は4月20日（日）14:00、参加費5,000円です。」
→ Claude がファイルを作成してデスクトップに保存してくれます。自分でコピペする必要なし。
うまく使うコツ
「あなたは〇〇の専門家です」と最初に役割を伝えると、回答の質がグッと上がります。
例：「あなたはプロモーションPDです。」「あなたはLP制作のプロです。」
プロジェクト機能を使おう
Claude Desktop の左サイドバーから「Projects」を作れます。「クライアントA用」「講座準備用」などプロジェクトを分けておくと、過去のやりとりが整理されて便利です。プロジェクトに資料ファイルを追加しておけば、毎回添付しなくても Claude が参照してくれます。

デザインの4つのルール

      
これだけ押さえれば「ダサい」は回避できます。

      

        

          

            
          

          
余白を恐れない

          
詰め込みすぎは読みにくさの原因。空白は「洗練」のサイン。要素間は広めにとる。

        


        

          

            
          

          
色は3色まで

          
メイン・サブ・アクセントの3色に絞る。多色使いはアマチュア感が出てしまう。

        


        

          

            
          

          
写真は明るく・鮮明に

          
暗い・ぼやけた写真は信頼感を下げる。顔が見える・明るい写真が最強の素材。

        


        

          

            
          

          
テキストは短く・端的に

          
LPは「読まれない」前提で作る。一文30文字以内を目安に。キャッチコピーはさらに短く。

Step 05 ／ 公開する

    
LPをネットに公開する

    
「GitHub（保管庫）」と「Vercel（サーバー）」の2つを使って、誰でもアクセスできるURLでLPを公開します。Claude Codeにお願いすれば、コマンドはほぼ自動で進みます。


    
    

      
📐
全体の流れ（イメージ）

      

        ネット公開には「ファイルを保管する場所」と「URLを発行してくれるサーバー」の2つが必要です。今回はそれぞれGitHubとVercelを使います。
      

      

        

          
💻

          
あなたのPC

          
Claude Codeで
LPを作る

        

        
→

        

          
🗄️

          
GitHub

          
＝ 保管庫
（Privateで保管）

        

        
→

        

          
🌐

          
Vercel

          
＝ サーバー
（公開URL発行）

        

      

      

        💡 一度この仕組みを作ってしまえば、あとは 「Claude Codeに『pushして』と頼むだけ」 で自動的にネット上のLPも更新されるようになります。
      

    


    
    

      
!
超重要：GitHubは必ず「Private」にしてください

      

        GitHubには Public（公開）とPrivate（非公開）の2種類があります。

        今回は必ず Private を選んでください。
      

      

        

          
❌ Public（公開）

          
中身のコードを世界中の誰でも見られる状態。Claude Codeとのやり取り・下書き・パスワード・お客様情報なども丸見えになる危険があります。

        

        

          
⭕ Private（非公開）

          
コード本体はあなたしか見られない状態。Vercel経由で公開する「LPの見た目」だけが、URLからアクセスする人に届きます。

        

      

      

        💬 「Privateにすると公開できないんじゃ？」と思いがちですが、VercelはPrivateリポジトリでも無料で公開URLを発行できます。安心してください。
      

    


    
    

      
0
事前準備（キックオフまでに済ませておく）

      

        

          
1

          

            
GitHubアカウントを作る

            

              github.com/signup からメールアドレスで作成。無料です。
            

          

        

        

          
2

          

            
Vercelアカウントを作る

            

              vercel.com/signup で「Continue with GitHub」を選ぶと、GitHubアカウントでそのままサインアップできます（同じメアドでOK）。
            

          

        

      

    


    
    

      
1
GitHubに「保管庫」を作る

      

        まずはあなたのLPを保管する箱（リポジトリ）をGitHub上に作ります。
      

      

        

          
1

          

            
GitHub右上の「＋」→「New repository」をクリック

            
ログイン後の画面、画面右上のプラスマークから新しいリポジトリ作成画面に進みます。

          

        

        

          
2

          

            
リポジトリ名を入力

            
英数字・ハイフンのみ。例：my-lp ／ サービス名-lp

          

        

        

          
3

          

            
⭕ Private を選ぶ（ここがいちばん大事！）

            
2つのラジオボタンから「Private」を選択。Publicは絶対に選ばないでください。

          

        

        

          
4

          

            
「Create repository」をクリック

            
下のボタンを押すと、空っぽのリポジトリができます。次のステップで中身を入れていきます。

          

        

      

    


    
    

      
2
LPのファイルをGitHubに送る（ターミナル不要）

      

        作ったLPのファイル（index.html など）をGitHubの保管庫に送ります。

        ターミナルは使いません。2つの方法があるので、ラクな方を選んでください。
      


      
      

        

          
A

          
github.com の画面から直接ドラッグ＆ドロップ（いちばんカンタン）

        

        

          ① さっき作った空のリポジトリページで 「uploading an existing file」 のリンクをクリック

          ② LPのフォルダをFinder/エクスプローラーで開く → 中身を全選択してブラウザにドラッグ＆ドロップ

          ③ 下の 「Commit changes」 ボタンをクリック

          ✨ これだけでGitHubに全ファイルが入ります。インストール不要。
        

      


      
      

        

          
B

          
GitHub Desktop を使う（更新が多い人向け）

        

        

          ① desktop.github.com からアプリをダウンロード → インストール → GitHubアカウントでログイン

          ② 「Add Local Repository」でLPのフォルダを選択（「git initしますか？」と聞かれたら Yes）

          ③ 左下の Summary欄に「初回コミット」 と入力 → 「Commit to main」

          ④ 右上の 「Publish repository」 → さっき作ったリポジトリ名で、Private のチェックを必ず入れて Publish

          ✨ 次回以降は「変更 → Commit → Push」の3クリックだけで更新できます。
        

      


      

        💡 迷ったらA：1回だけならブラウザのドラッグ＆ドロップが最短。何度も更新する予定なら B の GitHub Desktop が便利です。
      


      

        
CHECK

        

          GitHubのリポジトリページをリロードして、index.html などのファイルが表示されていればアップロード成功です ✨
        

      

    


    
    

      
3
Vercelに繋いで公開URLを発行する

      

        GitHubに保管したLPを、Vercelに「これを公開して」とお願いします。3クリックで公開URLが発行されます。
      

      

        

          
1

          

            
vercel.com/new を開く

            
Vercelにログインした状態で、新規プロジェクト作成画面を開きます。

          

        

        

          
2

          

            
「Import Git Repository」から先ほどのリポジトリを選ぶ

            
一覧にPrivateリポジトリも表示されます。先ほど作った my-lp を「Import」。
※ 初回は「VercelにGitHubへのアクセス権を渡す」確認が出ます → Authorizeで進めてください。

          

        

        

          
3

          

            
設定はそのまま「Deploy」をクリック

            
何も変更しなくてOK。HTMLだけのサイトはVercelが自動で認識してくれます。

          

        

        

          
✓

          

            
1〜2分待つと公開URLが発行されます

            

              https://リポジトリ名-xxxx.vercel.app

              これがあなたのLPの公開URLです。スマホからもアクセスできます！
            

          

        

      

    


    
    

      
4
あとはファイルを差し替えるだけで自動更新

      

        ここまで設定すれば、以降の更新はGitHubにファイルを上げ直すだけ。

        VercelがGitHubの更新を自動で検知して、数十秒で公開URL側にも反映されます。
      

      

        
UPDATE FLOW

        

          ① claude.ai でLPを修正してもらう → 新しいコードをコピペで保存

          ② GitHubの該当ファイルを開いて ✏️鉛筆マーク → 中身を貼り替えて「Commit changes」

          　（GitHub Desktop派なら Commit → Push の2クリック）

          ③ Vercelが自動でデプロイ → 公開URLが更新される（30秒〜1分）
        

      

    


    
    

      
+
独自ドメインをお持ちの方へ

      

        取得済みの独自ドメイン（例：yourname.com）をお持ちの方は、Vercelの公開URLに繋げることができます。
      

      

        

          
① Vercelプロジェクトの「Settings → Domains」で独自ドメインを入力

        

        

          
② 表示されるDNSレコードをドメイン会社の管理画面に設定

        

        

          
③ 数分〜数時間で反映。HTTPS（SSL）も無料で自動対応

        

      

      
ドメインをお持ちでない方は、Vercelの無料URLでまず公開 → 後から繋げることもできます。当日サポートしますので講師に声をかけてください。
