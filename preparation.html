Designer's Eye
プロのデザイナーはLPをこう見る

CCでLPを作る技術より大事なこと。それが「デザイナーの頭の中」です。当日の前に読んでおいてください。

「準備が9割」
CCで作れることは手段。
何を伝えるかが、すべてです。

どんなに綺麗なLPでも、伝えるべき情報が整っていなければ申込にはつながりません。当日CCに指示する前に、「何を・誰に・どう伝えるか」を決めておくことが最大の準備です。

1
アートとデザインは違う
Art / アート
自己表現。作者が伝えたいことを形にする。「好き」「美しい」が基準。見る人の解釈に委ねる。
Design / デザイン
目的達成のための設計。見る人に「行動してもらう」ことが基準。綺麗かどうかは手段でしかない。

LPはデザインです。「おしゃれに見せたい」は手段。「申し込んでもらう」「問い合わせしてもらう」が目的です。

2
情報をデザインするとは

LPに載せる情報は「全部」ではなく「必要なものだけ」。情報の取捨選択こそがデザインの第一歩です。

1
このLPで何をしてほしいか、1つ決める
「申し込んでほしい」「問い合わせしてほしい」「LINEに登録してほしい」。ゴールは必ず1つ。複数あると何もしてもらえません。
2
誰に届けたいかを1人に絞る
「全員に刺さるLP」は誰にも刺さりません。理想の1人を思い浮かべて書くと、読んだ人が「これ私のことだ」と感じます。
3
載せない情報を決める勇気を持つ
「全部説明しなきゃ」という気持ちはわかります。でも情報が多いほど読まれません。捨てることがデザインです。
3
だから準備が9割

CCはあなたの指示通りに作ります。指示の質＝LPの質です。

準備が足りないとき
「なんか違う」「イメージと違う」を繰り返す
→
準備できているとき
1回の指示で8割完成。あとは細かい調整だけ

ワークシートにしっかり記入することで、CCへの指示の質が上がります。タブ07「ワークシート」を丁寧に埋めてから、当日に臨んでください。

4
その上で、デザインを選ぶ

伝える内容が決まってはじめて、デザインの出番です。カラー・フォント・素材は「誰に届けたいか」から選びます。

Color
色はターゲットの「好みの世界観」から選ぶ。自分の好みではなく、お客さんが好きな色。
Font
フォントはサービスの「声のトーン」。高単価なら知的に。ママ向けならやわらかく。
Photo
素材はLPの印象を9割決める。暗い写真・ぼやけた写真は信頼を下げる。

Step 02
Claude Codeを使える状態にする

当日は「すぐ作業できる状態」でお越しください。このページを見ながら事前にセットアップを済ませておきましょう。

セットアップに不安がある方は、事前オンライン会で一緒に確認します。わからなくても大丈夫です。
1
Claude Codeとは？
What
ターミナル（黒い画面）で動くAIアシスタント。「指示するだけでファイルを作ってくれる」ツールです。
ChatGPTとの違い
ChatGPTは「答えを返す」だけ。Claude Codeはパソコン上のファイルを直接作ったり編集したりできます。
2
何ができるの？
LPや診断ツールをHTMLで作る — 日本語で「こんなページ作って」と言うだけで完成
色・フォント・レイアウトを修正する — 「もっとシックにして」「余白を広くして」でOK
複数ファイルをまとめて編集する — 1つの指示で関連するファイルを全部直してくれる
コードが書けなくても動く — プログラミング知識は一切不要
3
使う前に知っておくこと
費用がかかります（月額 約3,000円）
Anthropicの有料プラン（Max $20/月）が必要です。使い放題ではなく、使うほどクレジットを消費します。ワークショップ1日で大量に使っても、通常は$20の枠内に収まります。
ファイルが自動で変更されます
Claude Codeは指示した内容をファイルに直接書き込みます。「戻す」ボタンはないので、大きな変更の前はファイルをコピーして保存しておきましょう。
英語で返ってくることがあります
日本語で話しかけても、確認メッセージが英語になることがあります。「y」と入力してEnterを押せば進めます。困ったら「日本語で答えて」と追加してOK。

フォルダの考え方

Claude Codeは「今いるフォルダ」の中でだけ作業します。フォルダ＝作業部屋のイメージです。

フォルダ構成の例
workshop-lp/　　　　← プロジェクトフォルダ（作業部屋）
  ├─ lp.html　　　← Claude Codeが作るLPファイル
  ├─ profile.jpg　← 顔写真
  └─ banner.jpg　 ← その他の画像
まずデスクトップに 「workshop-lp」 というフォルダを1つ作る
使う写真はすべてそのフォルダの中に入れておく
Claude Codeを起動するときは そのフォルダに移動してから 起動する（当日一緒に確認します）
フォルダの外のファイルはClaude Codeから見えないので 1つのフォルダにまとめる のが大事
Glossary

知らなくてもワークショップには参加できますが、意味を知っておくと当日の理解が深まります。

ギットハブ
Web Service

Deploy
デプロイ
Web
作ったものをインターネット上に公開すること
「デプロイする」＝「ネットに上げる」「公開する」という意味です。今回はGitHub Pagesという機能を使ってデプロイします。デプロイが完了すると、スマホからでもURLでアクセスできるようになります。
Vercel
バーセル
Web Service
GitHubと連携して、自動でデプロイしてくれるサービス
GitHub Pagesと同じように「ネットに公開する」ためのサービスです。GitHubにファイルを保存すると自動でデプロイしてくれる点が便利。今回はGitHub Pagesを使いますが、慣れてきたらVercelも使ってみてください。無料枠あり。
Repository
リポジトリ
GitHub
GitHubの中の「プロジェクトフォルダ」のこと
パソコンの「フォルダ」と同じイメージです。1サービス1リポジトリで管理するのが基本。今回はLP用のリポジトリを1つ作って、そこにlp.htmlを入れます。「リポジトリを作る」＝「GitHubにフォルダを作る」と覚えてください。
Sandbox
サンドボックス
Claude Code
Claude Codeの「確認モード」。ファイルを変更する前に許可を求めてくれる
サンドボックスをONにすると、Claude Codeがファイルを書き換える前に「本当に実行していいですか？」と確認してきます。慣れないうちはONにしておくと安心です。慣れてきたらOFFにすると作業スピードが上がります。
.claudeignore
ドットクロードイグノア
Claude Code
「このファイルはClaude Codeに触らせない」と指定するファイル
プロジェクトフォルダの中に .claudeignore というファイルを作り、触らせたくないファイル名を書いておきます。パスワードや個人情報が入ったファイルは、念のためここに記載しておくと安心です。

デザインの4つのルール

      
これだけ押さえれば「ダサい」は回避できます。

      

        

          

            
          

          
余白を恐れない

          
詰め込みすぎは読みにくさの原因。空白は「洗練」のサイン。要素間は広めにとる。

        


        

          

            
          

          
色は3色まで

          
メイン・サブ・アクセントの3色に絞る。多色使いはアマチュア感が出てしまう。

        


        

          

            
          

          
写真は明るく・鮮明に

          
暗い・ぼやけた写真は信頼感を下げる。顔が見える・明るい写真が最強の素材。

        


        

          

            
          

          
テキストは短く・端的に

          
LPは「読まれない」前提で作る。一文30文字以内を目安に。キャッチコピーはさらに短く。

GitHub Collaboration

    
クライアントと
同じファイルをいじる方法

    
GitHubを使えば、離れた場所にいる人と同じファイルを安全に編集できます。この4コマで全体の流れを掴みましょう。


    
    

      
    


    
    

      

        
01

        
GitHubは「インターネット上の共有フォルダ」

      

      
DropboxやGoogleドライブに近いイメージです。ただしファイルの「変更履歴」も全部残ります。

      

        

          
普通のフォルダと同じこと

          
ファイルを入れる・取り出す・編集する。基本的な操作はフォルダと同じです。

        

        

          
変更履歴が残る

          
「誰が・いつ・何を変えたか」がすべて記録されます。間違えても前の状態に戻せます。

        

        

          
複数人で使える

          
Collaborator（共同作業者）として招待した人だけが編集できます。

        

        

          
無料で使える

          
基本機能は無料。プライベート（非公開）リポジトリも無料で作れます。

        

      

    


    
    

      

        
02

        
一緒に編集したい人を招待する

      

      
あやがやること

      

        

          
1

          

            
GitHubでリポジトリを開く

            
github.com → 対象のリポジトリ（例：gokigen-design/ai-workshop）を開く

          

        

        

          
2

          

            
Settings → Collaborators を開く

            
上部タブの「Settings」→ 左メニュー「Collaborators（協力者）」をクリック

          

        

        

          
3

          

            
「人を追加する」→ ユーザー名を入力

            
相手のGitHubユーザー名を入力して選択 → 「Add」をクリック

          

        

        

          
4

          

            
招待メールが相手に届く

            
相手のメールアドレスに招待が届きます。承認されると編集できるようになります。

          

        

      

    


    
    

      

        
03

        
招待を受け取ったら、自分のPCにコピーする

      

      
クライアント・チームがやること

      
「clone（クローン）」とは、GitHubのフォルダを自分のパソコンにコピーすることです。1回やるだけです。

      

        

          
1

          

            
招待メールを承認する

            
GitHubからのメールを開いて「Accept invitation」をクリック

          

        

        

          
2

          

            
ターミナルを開いてcloneする

            
以下のコマンドをコピペして実行。フォルダが自動で作られます。

            
# 共有フォルダを自分のPCにコピーする（1回だけ）
git clone https://github.com/gokigen-design/ai-workshop.git

          

        

        

          
3

          

            
Claude Codeでそのフォルダを開く

            
cloneしたフォルダをClaude Codeで開けば、すぐに編集できます。

          

        

      

    


    
    

      

        
04

        
変更したら「push」で全員に届ける

      

      
あや・クライアント共通

      
Claude Codeで編集したら、pushでGitHubに送ります。これが「保存して共有」の操作です。

      
      

        
      

      
# 変更したファイルを指定（preparation.htmlの場合）
git add preparation.html

# 「何を変えたか」のメモをつける
git commit -m "ワークシートの内容を更新"

# GitHubに送って全員に共有する
git push

    


    
    

      

        
05

        
コマンド早見表

      

      
この4つだけ覚えれば大丈夫。迷ったらここに戻ってきてください。

      

        

          git clone
          GitHubのフォルダを自分のPCにコピーする（初回1回だけ）
        

        

          git add
          保存するファイルを選ぶ
        

        

          git commit
          「何を変えたか」のメモをつけて記録する
        

        

          git push
          GitHubに送って全員と共有する　← 一番大事！
        

      

    


    
💡 Claude Codeを使えば、これらのコマンドは「git push して」と話しかけるだけで自動で実行してくれます。難しく考えすぎなくて大丈夫！
